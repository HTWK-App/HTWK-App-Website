Client-Server Architektur

Die HTWK-App setzt auf die altbekannte Client-Server Architektur. Die App ist der Server und der Server der Client....äh, natürlich andersherum ;)

Webseite

Die App wurde als Webseite entworfen und setzt samit auf HTML, CSS und Javascript. Damit können wir den Studenten und Mitarbeitern der HTWK eine moderne Oberfläche präsentieren, die leicht umgebaut werden kann. Wir benutzten JQuery und Javascript für die Logik, JQueryMobile für die UI und FontAwesome für Icons. Daten, die die App anzeigt, werden immer brandneu und unter Nutzung neuster Technologien vom Server bezogen.

Server

Die Server-Anwendung besteht aus mehreren Microservices, die vornehmlich in Java und Scala geschrieben sind. Unter Java wird das Spring-Framework, unter Scala das Play-Framework benutzt. Der Server bieten eine HTTP-Rest Schnittstelle an, von der JSON formatierte Daten bezogen werden können.

App

Das Framework Apache Cordova transformiert die Webseite in eine App, die in alle gängigen App-Stores gestellt werden kann. Für den Google-Play Store haben wir das schon gemacht, für alle anderen Suchen wir noch Möglichkeiten...(ganz ehrlich Apple...100$ pro Jahr für ein Non-Profit Projekt?!).

Under the Hood wird ein Browser auf deinem Smartphone gestartet, der die Webseite lädt. Nur die Bedienelemente sind ausgeblendet.

Was ist derzeit in Planung?

Derzeit gibt es noch einige kleinere Bugs in der App. Diese werden wir mit Nachdruck ausmerzen, damit die App auf so vielen Geräten wie möglich fehlerfrei funktioniert.

Querverweise innerhalb der App werden demnächst kein Problem mehr sein. Damit findest du noch schneller die Informationen, die du brauchst!

Freiheit ist das größte Gut!

Und genau nach diesem Grundsatz ist auch die App entstanden.
