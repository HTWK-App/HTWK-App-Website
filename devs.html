Client-Server Architektur

Die HTWK-App setzt auf die altbekannte Client-Server Architektur. Die App ist der Server und der Server der Client....äh, natürlich andersherum ;)

Webseite

Die App wurde als Webseite entworfen und setzt damit auf HTML, CSS und Javascript. Damit können wir den Studenten und Mitarbeitern der HTWK eine moderne Oberfläche präsentieren, die leicht umgebaut werden kann.

Wir benutzten JQuery und Javascript für die Logik, JQueryMobile für die UI und FontAwesome für Icons. Daten, die die App anzeigt, werden immer brandneu und unter Nutzung neuster Technologien vom Server bezogen.

Server

Die Server-Anwendung besteht aus mehreren Microservices, die vornehmlich in Java und Scala geschrieben sind. Unter Java wird das Spring-Framework, unter Scala das Play-Framework benutzt. Der Server bieten eine HTTP-Rest Schnittstelle an, von der JSON formatierte Daten bezogen werden können.

App

Das Framework Apache Cordova transformiert die Webseite in eine App, die in alle gängigen App-Stores gestellt werden kann. Für den Googles Play-Store haben wir das schon gemacht, für alle anderen suchen wir noch Möglichkeiten...(ganz ehrlich Apple...100$ pro Jahr für ein Non-Profit Projekt?!).

Under the Hood wird ein Browser auf deinem Smartphone gestartet, der die Webseite lädt. Nur die Bedienelemente des Browsers fehlen.

Die App

Wie schon erwähnt ist die App eigentlich eine Webseite, die in jedem Browser aufgerufen werden kann (bspw. hier). Je Menüpunkt innerhalb der App existiert eine HTML-Datei, die das DOM und das dazugehörige Script kapselt. Allgemeine Funktionen und Libraries wurden in einzelne Javascript Dateien ausgelagert.

Die gesamte Webseite wurde anhand der Vorschläge des Google Pagespeed Plugins optimiert. Trotzdem besteht hier noch einiger Bedarf. Gerade beim Starten der App geht alles recht zäh zu.
